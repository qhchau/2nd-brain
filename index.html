2ND BRAIN ARCHITECTURE

WHAT YOUR AI IS BUILDING FOR YOU

A living knowledge base that grows every day. You feed it raw input — it synthesizes, cross-links, and remembers everything.

DIRECTORY STRUCTURE
2nd-brain/
├── CLAUDE.md           ← schema, conventions, AI rules
├── raw/                ← YOU write here
└── wiki/               ← AI maintains this
    ├── .system/          ← index + log
    ├── clients/          ← auto-created
    ├── projects/         ← auto-created
    ├── people/           ← auto-created
    └── ...               ← grows as you ingest
2nd-brain/
├── CLAUDE.md           ← schema, conventions, AI rules
├── raw/                ← YOU write here
├── wiki-INDEX.md       ← AI maintains this
├── wiki-LOG.md         ← AI maintains this
└── wiki-<slug>.md      ← one file per entity, flat
INPUT LAYER
raw/
You write here. Save Copilot output, meeting notes, PDFs. AI reads and never modifies it. This is your audit trail.
KNOWLEDGE LAYER
wiki/
AI-maintained knowledge base. Every entity — client, project, person, tool — gets its own linked page. Updated on every ingest.
RULEBOOK
CLAUDE.md
Schema, naming conventions, ingest and lint workflows. Edit this to change how the AI behaves.

THE THREE RULES
raw/
You write, AI reads. Never delete source files. Everything the AI knows traces back here — it's your audit trail.
wiki pages
AI writes, you read. Don't manually edit these pages — they'll be overwritten on the next ingest.
CLAUDE.md
The rulebook. Edit this file to change how the AI ingests, classifies, and cross-links your data. It's the only config file you ever need to touch.

STEP 04 / INGEST

        
LOAD INTO AI & START ASKING

        
Open your AI (PASA AI or Claude Code) from your 2nd-brain folder, then run the ingest command.


        
INGEST COMMAND

        

          ingest what's new in the raw folder
          
        


        

          
After ingestion, ask anything — project status, action items, decisions, who owns what.

STEP 05 / END OF DAY

        
CLEAN UP THE WIKI

        
Before you close, run the lint command in PASA AI or Claude Code. It keeps the wiki tidy and accurate.


        
RUN IN PASA AI OR CLAUDE CODE

        

          lint the wiki to clean up
          
        


        

          
WHAT THIS DOES

          
Fixes broken cross-links  ·  Removes duplicate entries  ·  Tidies page formatting  ·  Rebuilds the master index
